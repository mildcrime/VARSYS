# VARSYS — Руководство пользователя (Русский)

[English version](USAGE_EN.md)

> Только авторизованное тестирование и обучение. См. [DISCLAIMER](../DISCLAIMER.md).

## 1. Основы

- **Загрузка:** появляется анимированный логотип VARSYS, затем домашняя карусель.
- **Навигация:** вращайте энкодер для перемещения между иконками; центральная —
  выбранная. **Нажатие** энкодера — открыть. **Кнопка «назад»** (сбоку) или
  **долгое нажатие** — вернуться / прервать текущее действие.
- **Статус‑бар** (вверху справа): время и заряд.
- **Настройки:** Дом → *Настройки* — язык (RU/EN), тёмная тема, звук, вибро,
  яркость, **режим Эксперт**, О системе.

## 2. Sub‑GHz (CC1101)

Дом → **Sub‑GHz**. Слева — частота и живой RSSI; справа — список действий
(прокрутка энкодером):

- **Частота** — перебор пресетов (315 / 390 / 433.92 / 434.42 / 868.35 / 915 МГц).
- **Скан** — свип по пресетам, переход на самый сильный.
- **Запись** — захват сырого OOK; декодер показывает протокол + ключ.
- **Воспроизвести** — повторить последний захват.
- **Сохранить** / **Библиотека** — сохранить (`.sub`) / просмотр и повтор записей.
- **Спектр** — живой свип RSSI вокруг текущей частоты.
- **Перебор** — открывает экран перебора (см. ниже).

### Перебор (шлагбаумы / фиксированные коды)

1. Сначала задайте частоту шлагбаума в Sub‑GHz (обычно 433.92 или 315 МГц).
2. Sub‑GHz → **Перебор**. Выберите **Протокол** (Came / Nice / Ansonic /
   Holtek / Linear / Chamberlain или **All**) и **Повторы** (2–3).
3. **Старт.** Передаются полные кадры по каждому коду. Когда шлагбаум открылся —
   нажмите **«назад»**: последние ~48 кодов сохранятся как кандидаты в
   `/brute/found_*.txt`.
4. **Candidates** — автопрогон сохранённого окна по одному с паузой и живым
   индексом. Когда снова открылось — нажмите **«назад»**: точный подтверждённый
   код сохранится в `/brute/confirmed_*.sub`.

## 3. Инфракрасный порт

Дом → **Infrared**: **Захват** пульта, **Воспроизвести**, или **Выключить ТВ**
(серия распространённых power‑кодов).

## 4. NFC / RFID (PN532)

Дом → **NFC**: **Читать** метку (показывает UID/тип), **Сохранить** UID,
**Write** — запись блока Mifare Classic (ключ A по умолчанию). Нужны модуль
PN532 и метка.

## 5. WiFi

Дом → **WiFi** — скан точек доступа. Выберите AP и нажмите для включения
**deauth**; во время деаута **детектируются handshake (EAPOL)** и считаются
(также `wifi hs` в CLI). **Evil portal** (captive‑точка для авторизованных
фишинг‑симуляций) — в гейтнутом разделе **Эксперт**.

## 6. Bluetooth / NRF24 / GPS / iButton / FM

- **Bluetooth** — скан/разведка BLE; нажатие на запись показывает её адрес.
- **NRF24** — анализатор активности каналов 2.4 ГГц (нужен модуль NRF24).
- **GPS** — спутники, фикс и координаты (нужен GPS‑модуль на UART).
- **iButton** — чтение ROM ключа Dallas (нужен 1‑Wire пробник), сохранение.
- **FM** — выбор частоты и включение маломощной передачи (Si4713).

## 7. BadUSB

Дом → **BadUSB**: подключите устройство к хосту по USB; нажмите для запуска
демо Ducky‑скрипта. Полные скрипты кладутся в хранилище (`/ducky/`).

## 8. WebUI + OTA

Дом → **WebUI** → **Старт**. Подключитесь к точке доступа `VARSYS‑XXXX`
(пароль `varsys1234`), откройте `http://192.168.4.1`:
- статус устройства, **библиотека сигналов** (просмотр / скачивание `.sub`),
- **OTA** — загрузка нового `firmware.bin` для обновления по воздуху.

## 9. Serial CLI

По USB‑Serial (115200): `help`, `ver`, `freq <кГц>`, `rssi`, `rec`, `replay`,
`ir tvoff`, `wifi scan`, `wifi hs`, `ble scan`, `reboot`.

## 10. Раздел «Эксперт»

Виден только при включённом **режиме Эксперт** в настройках. Содержит:
- **Dev tools** — система (heap/PSRAM/uptime/температура), I²C‑сканер,
  дамп регистров CC1101, factory reset.
- **Captive portal** — авторизованная фишинг‑симуляция.
- **Глушилка / BLE‑spam** — **нерабочие заглушки** (см. Disclaimer).

## Файлы в хранилище

- `/signals/*.sub` — сохранённые записи Sub‑GHz
- `/brute/*.txt`, `/brute/confirmed_*.sub` — кандидаты / подтверждённый код перебора
- `/nfc/*.txt`, `/ibutton/*.txt` — сохранённые идентификаторы
- `/portal_creds.txt` — данные с captive‑портала

Используется SD‑карта при наличии, иначе внутренняя LittleFS.
